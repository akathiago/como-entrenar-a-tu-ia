## 1.3 · Comandos y atajos

- **Tiempo:** 20 minutos

> **Lección interactiva:** escribí `/clase-1-3` en Claude Code

## Los slash commands

Los comandos que empiezan con `/` son órdenes directas para Claude Code (no para la conversación). Escribí `/` y aparece el menú completo. Estos son los que valen la pena saber:

### Los que vas a usar todos los días

| Comando | Qué hace |
|---------|----------|
| `/clear` | Conversación nueva, de cero (los archivos quedan, el chat se borra) |
| `/resume` | Retoma una conversación anterior después de cerrar |
| `/help` | Lista de todos los comandos |

### Sesión y contexto

| Comando | Qué hace |
|---------|----------|
| `/compact` | Comprime la conversación cuando se pone muy larga (libera memoria de trabajo) |
| `/context` | Muestra qué archivos tiene presentes Claude ahora |
| `/usage` | Cuánto de tu cuota mensual llevás usado |
| `/model` | Cambia de modelo (más rápido vs más inteligente) |

### Configuración y salud

| Comando | Qué hace |
|---------|----------|
| `/doctor` | Chequeo general: qué anda y qué no |
| `/permissions` | Qué puede hacer Claude sin pedirte permiso |
| `/init` | Crea el archivo de memoria del proyecto (lo vemos en la próxima lección) |
| `/quit` | Salir de Claude Code |

## El botón de pánico: Escape

Apretá **Escape** y Claude frena en seco, en el medio de lo que esté haciendo.

Usalo sin culpa: si ves que agarró para cualquier lado, frenalo, aclarale y seguí. Siempre tenés el volante.

## Rebobinar: Escape dos veces

Apretá **Escape dos veces** y aparecen las opciones de rewind:

- Volver la conversación atrás (los archivos quedan como están)
- Volver la conversación Y los archivos atrás (deshace los cambios)

Es viaje en el tiempo. Ideal para experimentar tranquilo: probá cosas, y si no te gusta, rebobiná.

## Pedirle que piense más

Hay palabras que activan más capacidad de razonamiento:

| Palabra | Efecto |
|---------|--------|
| `think` | Pensá un poco más |
| `think harder` | Análisis más profundo |
| `ultrathink` | Máxima potencia (aparece un indicador arcoíris, en serio) |

Sirven para decisiones complejas: *"think harder: ¿cómo estructurarías este informe para la dirección?"*

> **Ojo:** pensar más gasta más cuota. Para pedidos simples no hace falta.

## Los tres modos de trabajo

Con **Shift + Tab** vas rotando entre tres modos:

| Modo | Comportamiento | Cuándo usarlo |
|------|---------------|---------------|
| **Normal** | Te pide permiso antes de cada cambio | Mientras aprendés. Es el default. |
| **Auto-accept** | Hace todo sin preguntar | Cuando ya confiás en la tarea |
| **Plan** | Primero arma un plan, vos lo aprobás, después ejecuta | Tareas grandes con muchos pasos |

El modo Plan es el secreto mejor guardado para no programadores: le pedís algo grande, te muestra el plan en castellano, corregís lo que haga falta y recién ahí arranca.

## Comandos propios

Los comandos `/clase-1-1`, `/clase-1-2` del taller no vienen con Claude Code: son archivos de texto en la carpeta `.claude/commands/`. Cualquiera puede crear los suyos.

¿Un informe que armás todas las semanas? `/informe-semanal`. ¿El formato de tus minutas? `/minuta`. En la Clase 4 creás los tuyos. Adelanto: se lo podés pedir a Claude directamente ("creame un comando que...").

## Chuleta de atajos

| Atajo | Acción |
|-------|--------|
| `Escape` | Frenar a Claude |
| `Escape × 2` | Opciones de rebobinado |
| `Shift + Tab` | Cambiar de modo |
| `Ctrl + V` | Pegar imágenes (¡también en Mac es Ctrl, no Cmd!) |
| `/` | Menú de comandos |
| `@` | Referenciar archivos |

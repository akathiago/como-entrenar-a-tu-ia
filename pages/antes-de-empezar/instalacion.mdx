## Instalación

- **Tiempo:** 15 minutos
- **Dificultad:** copiar y pegar un comando. Posta.

## Antes de arrancar, necesitás

- **Suscripción a Claude Pro o Max** (la versión gratis NO alcanza)
  - Si no la tenés, creala en [claude.ai](https://claude.ai). Pro cuesta USD 20 por mes.
- Windows 10 o más nuevo, Mac (macOS 10.15+) o Linux
- Conexión a internet
- **Ningún programa de programación.** Nada de instalar cosas raras antes.

## Paso 1: abrí la terminal

La terminal es la ventana donde vas a escribir comandos. Se abre así:

**Windows** (el camino que usa la mayoría del taller):
1. Apretá la tecla de Windows
2. Escribí "PowerShell"
3. Enter

**Mac:**
1. Apretá `Cmd + Espacio` para abrir Spotlight
2. Escribí "Terminal"
3. Enter

Te va a aparecer una ventana con un cursor titilando. No muerde. Ahí se escribe.

## Paso 2: instalá Claude Code

Copiá el comando que corresponde a tu sistema, pegalo en la terminal y apretá Enter.

**Windows (PowerShell):**
```powershell
irm https://claude.ai/install.ps1 | iex
```

**Mac / Linux:**
```bash
curl -fsSL https://claude.ai/install.sh | bash
```

Cuando termine, **cerrá la terminal y abrila de nuevo.** Este paso parece tonto pero es clave: la terminal necesita reiniciarse para enterarse de que Claude existe.

## Paso 3: verificá que se instaló

En la terminal nueva, escribí:

```bash
claude --version
```

Si te muestra un número de versión, listo. Si te dice algo tipo "no se reconoce el comando", andá a la sección de problemas acá abajo.

## Paso 4: iniciá sesión

1. Escribí `claude` y apretá Enter
2. Elegí **"Claude account with subscription"** (NO "Anthropic Console", esa es otra cosa)
3. Se te va a abrir el navegador para iniciar sesión: hacelo con tu cuenta de claude.ai
4. Elegí el estilo de texto que más te guste
5. Si ves la pantalla de bienvenida de Claude Code, ya está

## Si algo salió mal

### "El término 'claude' no se reconoce..." (Windows)
- Cerrá PowerShell del todo (todas las ventanas) y abrilo de nuevo
- Si sigue igual, reiniciá la computadora. Sí, el clásico. Funciona.

### "command not found: claude" (Mac)
- Cerrá la terminal y abrila de nuevo
- Si sigue igual, escribí `source ~/.zshrc` y probá otra vez

### "Necesitás una suscripción"
- Entrá a [claude.ai](https://claude.ai) y verificá que tu suscripción Pro esté activa
- Cerrá sesión y volvé a entrar

### "Permission denied" o error de permisos (Windows)
- Cerrá PowerShell, buscalo de nuevo en el menú de inicio, click derecho y "Ejecutar como administrador", y repetí el comando de instalación

### El diagnóstico automático
Una vez adentro de Claude Code, escribí `/doctor`: te muestra qué anda y qué no, como un chequeo médico.

### Nada de esto funcionó
Traé el error a clase o mandalo por el grupo del taller. Sacale captura de pantalla a la terminal (o copiá el texto del error): con eso alcanza para diagnosticar.

## Chuleta de comandos

| Comando | Qué hace |
|---------|----------|
| `claude` | Abre Claude Code |
| `claude --version` | Muestra qué versión tenés |
| `claude doctor` | Diagnóstico desde la terminal |
| `/doctor` | Diagnóstico adentro de Claude Code |
| `/help` | Lista de comandos disponibles |

## Para la Clase 3: tres cosas más

Con Claude Code alcanza para las Clases 1 y 2. La Clase 3 (construir y publicar) usa tres herramientas más. **Instalalas antes de esa clase**, no durante: son diez minutos que no querés gastar en el medio de la construcción.

| Qué | De dónde | Para qué |
|-----|----------|----------|
| **Node.js** | [nodejs.org](https://nodejs.org) (versión LTS) | Construir el sitio y verlo en tu compu |
| **Git** | [git-scm.com](https://git-scm.com) | Llevar el historial de cambios del proyecto |
| **GitHub CLI** | [cli.github.com](https://cli.github.com) | Que Claude suba el proyecto sin que escribas comandos |

Y una **cuenta de GitHub** ([github.com](https://github.com), gratis): no se instala, se crea.

Para verificar que quedaron, escribí esto en la terminal. Cada uno tiene que responder con un número de versión:

```
node --version
git --version
gh --version
```

Si alguno dice "command not found" o "no se reconoce", ese falta. La lección 3.1 te lo chequea igual al arrancar, así que si algo se escapa no es el fin del mundo: solo vas a perder tiempo de clase.

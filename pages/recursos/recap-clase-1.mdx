# Recap · Clase 1 · COMPRENDER

Para releer en 5 minutos después de la clase.

## Lo esencial

1. **Dirigís, no ejecutás.** Describí el resultado que querés; el cómo es problema de Claude.
2. **Trabajá viendo.** Terminal a un lado, editor (VS Code) al otro. Nunca a ciegas.
3. **@ es señalar.** `@archivo.md` un archivo, `@carpeta/` todos los archivos de la carpeta.
4. **La tecla ESC es el volante.** Una vez frena, dos veces rebobina. Experimentá sin miedo.
5. **CLAUDE.md es la constitución.** Lo que Claude tiene que saber siempre, escrito una vez.

## Los comandos que quedaron instalados en tu cabeza

| Comando/atajo | Para qué |
|---------------|----------|
| `claude` | Abrir Claude Code (en la carpeta correcta) |
| `@` | Referenciar archivos y carpetas |
| `/clear` | Conversación nueva |
| `/resume` | Retomar donde quedaste |
| `ESC` / `ESC ×2` | Frenar / rebobinar |
| `Shift + Tab` | Cambiar de modo (normal / auto / plan) |
| `/init` | Crear el CLAUDE.md del proyecto |

## El prompt de la clase

```
Mirá @caos-heredado/ y decime qué hay: qué archivos, de qué se tratan,
y por dónde empezarías a ordenar este desastre.
```

## Tarea para la semana

1. Instalá VS Code si no lo hiciste en clase
2. Abrí Claude Code en una carpeta TUYA con archivos reales (una copia, si te deja más tranquilo)
3. Pedile que te diga qué hay y que te proponga un orden
4. Escribí tu CLAUDE.md global (`~/.claude/CLAUDE.md`): quién sos, a qué te dedicás, cómo te gusta que te escriban

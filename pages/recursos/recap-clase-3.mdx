# Recap · Clase 3 · PUBLICAR

Para releer en 5 minutos después de la clase.

## Lo esencial

1. **El loop:** planificar → construir → iterar → guardar → publicar. Sirve para cualquier proyecto.
2. **Requisitos primero.** La entrevista con Claude termina en `REQUISITOS.md`: el contrato desde el que se construye.
3. **localhost es ensayo,** la URL de Vercel es el estreno. `localhost:3000` no se puede compartir.
4. **Iterar con capturas:** Ctrl+V con la imagen marcada le gana a cualquier descripción.
5. **Los errores se copian, no se interpretan.** Texto rojo en la terminal → directo a Claude.

## Vocabulario que ya es tuyo

| Término | Qué es |
|---------|--------|
| **Servidor local / localhost** | Tu compu mostrándote el proyecto en privado |
| **Repo / commit / push** | Carpeta con historial / foto del momento / subir a la nube |
| **Deploy** | Publicar en internet |
| **Auto-deploy** | Push a GitHub → Vercel republica solo |
